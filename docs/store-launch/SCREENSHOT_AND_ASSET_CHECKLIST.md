# Screenshot And Asset Checklist

Use screenshots from the exact build being submitted. Do not show features that are not active, wired, or approved for launch.

## Required Assets

- [ ] App icon.
- [ ] iOS screenshots for required iPhone sizes.
- [ ] iOS iPad screenshots if the app supports iPad.
- [ ] Android phone screenshots.
- [ ] Android tablet screenshots if supported.
- [ ] Google Play feature graphic.
- [ ] Optional promo graphic or preview video if appropriate.

## Core Screens To Capture

- [ ] Home or primary dashboard.
- [ ] Main navigation.
- [ ] Account/profile state.
- [ ] Core app-specific workflow.
- [ ] Settings, support, or account controls.
- [ ] Empty/loading/error state if useful.
- [ ] Permission prompt context if relevant.

## Screenshot Rules

- Use real, review-safe demo data.
- Avoid secrets, real private user data, private emails, API keys, addresses, or payment details.
- Avoid screenshots implying live payments, dispatch, brokerage, emergency support, medical advice, legal advice, financial advice, verified transactions, or official partnerships unless confirmed.
- Match screenshots to store copy.
- Keep status bars, device frames, and overlays consistent.
- Keep text readable on mobile.
